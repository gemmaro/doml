DOML - Document Overridable Markup Language

DOML is one of the structural and lightweight markup language for document. As it has overridable in its name, you can define and modify markup definitions for any documents.

A DOML document consists of a preamble definition and content. Preamble definition and content are separated by a blank line. The content consists of multiple definitions or blocks. The block consists of several inlines. The beginning of this document in DOML is as follows. 

(# #): title
(* *): emphasis

(# DOML - Document Overridable Markup Language #)

DOML is one of the structural and lightweight markup language for document.
As it has overridable in its name, you can define and (*modify*) markup definitions for any documents.

The definition section defines the block or inline syntax. Block or inline syntax is defined as the following format in one line. 

left-tag right-tag: category

Tag notation must be symmetric. Only ASCII characters may be used. In particular, the outermost character must be the one that has a symmetric pair. That is: ( ), { }, [ ], and < >.

Category defines the meaning given to tags. The category set and any meaning or interpretation is up to the processor and is not defined in this document.

Definition can be overridden in the middle of the document. You must define the syntax for the definition in the preamble.

[! !]: definitions
